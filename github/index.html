Git commandments

A quick guide to using git & github

  1. install Git
    • download from here
  2. user configuration (run commands in Git Bash):
    • git config --global user.name to see configured user
    • git config --global user.email to see user email
    • git config --global user.name your_name to set/update user name
    • git config --global user.email your_email to set/update user email
  3. create a local git repository:
    • open or navigate power shell/terminal to projects folder and run the commands:
    • git init initialises folder as repository
    • git status displays status for current repo
    • git add "filename" or git add . to add all untracked files
    • git commit -m "message" (to commit the saved changes with a descriptive -m message)
  4. copy repository online:
    • go to github.com > repositories > new
    • name it same as the project folder
    • run commands shown under this line:
      "...or push an existing repository from the command line"
      (type them if copy/pasta erros out)
    • if it fucked up, delete the .git folder from the projects folder and restart at step 3
    • projects should now be visible on github.com account
  5. download repository from github to a projects folder:
    • run terminal in the folder where the new projects will be stored
      (preferably a new folder)
    • git clone >link of github repo, ending in .git<
  6. add new changes to github:
    • git status
    • git add .
    • git commit -m "message to git log about the changes made in code"
    • git push will save the stuff to github
  7. download code updates or changes from github:
    • git pull
  8. to cancel any local changes made since last commit:
    • git checkout . (notice the space and the period)
  9. merging
    • to pull & merge, and overwrite any local code differences:
    • git merge --strategy-options theirs
    • to pull & merge the code from github but use the local code if there are differences:
    • git merge --strategy-option ours
  10. branching
    • git branch shows current branch, marked with an *
    • git branch >name< creates a new branch with selected name
    • git checkout >name< switches to the new branch
    • git checkout -b >name< creates and switch to a new branch in one step
  11. managing branches
    • after pushing code to a branch a new button will appear on github's project page:
      Compare & pull request
    • clicking the button will display changes
      & a text box to describe the new code along with a new button:
      Create pull request
    • after creating the pull request a new option becomes available:
      Merge pull request
    • from there it gets analized/compared by someone else in the team
    • git conflicts coming soon
  12. other git commands
    • git diff to see any changes made since last commit
